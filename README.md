# MsgManager

一、	需求分析：
一个包含成绩信息的学籍管理信息系统，应该要实现以下功能：
1.	教师能够录入学生资料
2.	教师能够录入对某一场考试进行成绩登记
3.	教师能够查看所有学生的学籍信息和成绩信息，并且可以按某些信息（如学号，姓名，成绩等）进行排序
4.	学生能够查询任意一场考试中自己的成绩
5.	学生权限低于教师，学生无法查看其他人的成绩，无法对信息进行修改
6.	所有数据均能保存
7.	简易明了的UI
